分片哈希+分片上传

发表于 2026-04-15 分类于 技术复盘

之前负责的网络中控平台里面有个功能：终端升级。顾名思义，就是在平台上传不同型号的固件包、软件包等，最终下发到教室更新。

有天测试同学联系到我，说上传包完成后报错了去看看。OK，F12 打开调试工具，就看到了一条 error 信息：

getFileHash: NotReadableError

The requested file could not be read, typically due to permission problems that have occurred after a reference to a file was acquired.

初看好像是没有权限访问，无法读取文件，但是细想一下，只有这个文件上传成功后获取文件 md5 值报错，其他文件都是成功状态，猜测问题应该出在这个文件上面。将现有拿到的信息去问 ChatGPT， AI 给出的常见原因有：

  1. 用户权限或文件正在被占用
  2. File/Blob 对象已经失效或不是合法的 File
  3. 浏览器安全策略（比如在某些上下文或隐私模式下）
  4. 读入方式或读取时机不当
  5. 浏览器兼容或实现问题
  6. 文件大小或资源受限

咦，文件大小也有影响？！好吧，当即去看看文件大小有 3G 多，再去看看 getFileHash 的实现

猜测因为文件大小过大，所以一次性读取文件报错了。

幸好，AI 也给出了解决方法：

尝试读取较小文件验证；如果大型文件，考虑按分片读取（file.slice）并对每片计算哈希（例如采用 spark-md5 分片增量哈希），而不是一次性把整个文件读入内存
