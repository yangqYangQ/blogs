​        年初接手了同事开发的一个AI分析报告的功能，在此基础上，需要实现勾选任意预设指标来生成对应指标的分析报告功能。好的，话不多说，先来熟悉下之前的代码，梳理下需求。经过两天的挣扎，发现因为报告还需要下载成每页 A4 尺寸的 PDF，所以之前的同事把每页的内容都固定了，比如第一页内容是 [模块1，模块2]，第二页的内容是 [模块2.1] ...，很显然如果现在要改为勾选任意指标来生成报告的话，之前的这种写法很明显不符合要求了。于是开始了第一步拆分模块。

#### 1. 拆分模块

之前的分析报告总共有 34 页，每页有若干个模块，于是开始了漫漫拆分之路。

梳理需求发现分析指标总共有4个一级指标，17个二级指标，37个二级指标，整体是一个树形结构。

```javascript
// 分析指标
const reportModules = [
    {
        // 一级指标
        id: '1',
	label: 'xx1',
        children: [
            {
                // 二级指标
                id: '1-1',
                label: 'xx1-1',
                children: [
                    // 三级指标
                    {id: '1-1-1', label: 'xx1-1-1'},
                    ...
                ]
            },
            ...
        ]
    },
    ...
]
```

定义变量 `selectedReportModuleIds` 保存勾选的指标 `id` ，模板里拆分模块的结构如下：

```vue
<div class="original-report-structure">
    <!-- 使用 v-show 是考虑到在第一次调用接口后分析报告内容就不会再变了，后续只是显示与否切换 -->
    <div v-show="selectedReportModuleIds.includes('1')" class="module module-1">
        <!-- 这里是模块内容 -->
    </div>

    <div v-show="selectedReportModuleIds.includes('1-1')" class="module module-1-1"></div>
    ...
</div>
```

因为每页的高度固定，所以当一个模块内容高度超过一页高度时，需要将其拆分为多个子模块

```vue
<!-- 总模块内容高度过高，拆分成多个子模块 -->
<div v-show="selectedReportModuleIds.includes('2-1-1')">
    <div class="module module-2-1-1">...</div>
    <div class="module module-2-1-1">...</div>
    ...
</div>
```

好的，至此所有模块已经拆分完毕！



#### 2. 计算分页后的模块

拆分后的模块渲染完毕后即可计算分页模块。遍历所有的 `module` 类，调用 `getBoundingClientRect`  获取精确高度（`px`），已知每页内容区域高度阈值，那么就可以算出每页存放哪些 `modules` ，将其存放在变量 `pagedModules`

```javascript
    groupModules() {
      const modules = this.$refs.originalReportStructure?.querySelectorAll?.('[class^="module"]')

      let currentGroup = []
      let currentHeight = 0

      modules.forEach(module => {
        // 使用 getBoundingClientRect 获取精确高度（px）
        const rect = module.getBoundingClientRect()
        const height = rect.height

        if (height === 0) return

        // 判定是否添加到当前组 （thresholdPx：每页内容区域高度阈值）
        if (currentHeight + height <= thresholdPx) {
          // 使用 cloneNode 克隆模块并添加到当前组
          currentGroup.push(module.cloneNode(true))
          currentHeight += height
        } else {
          // push 当前组（可能为空，但我们避免 push 空组）
          if (currentGroup.length > 0) {
            this.pagedModules.push(currentGroup)
          }

          // 开始新的组并添加当前模块
          currentGroup = [module.cloneNode(true)]
          currentHeight = height

          // edge: 如果单个模块本身就大于 thresholdPx(理论上这个时候就应该再细分模块)，这个模块将单独成组（上面已经处理）
        }
      })

      // 添加最后一组（如果有未存储的模块）
      if (currentGroup.length > 0) {
        this.pagedModules.push(currentGroup)
      }
    }
```



#### 3. 报告重新排版布局

拿到 `pagedModules` 后即可以渲染排版后的报告

```vue
<!--排版后的报告-->
<div class="formatted-report-structure">
    <div v-for="(page, index) in pagedModules" :key="index" class="report-page">
    	<div v-for="(module, moduleIndex) in page" :key="moduleIndex" v-html="module.outerHTML"/>
    </div>
</div>
```

#### 4. 处理 `echarts` 图表

回到第一步拆分模块，处理图表

```vue
<!-- 图表类的结构保持相同，使用 echarts-module 类标记，后面拼接的是 moduleId -->
<div class="module module-2-2-3 echarts-module-2-2-3">
  <div class="chartBox">
    <!-- dofar-charts 是封装的一个公共组件类 -->
    <dofar-charts
      ref="chart2"
      :chart-data="option1"
      :height="'0.9896rem'"
      :width="'100%'"
    />
  </div>
</div>
...
```

来到第三步重新排版布局时，发现每个图表都需要且有可能不同的属性有 `chartData` 、`height` 、`width` ，因此定义变量 `moduleChartMap` 保存每个图表的属性值

```javascript
const moduleChartMap = {
    // [echarts-module key]: [chartData, width, height]
    '2-2-3': [this.option1, '100%', '0.9896rem'],
    ...
}
```

接下来可以重新排版了，调整代码

```vue
<!--排版后的报告-->
<div class="formatted-report-structure">
    <div v-for="(page, index) in pagedModules" :key="index" class="report-page">
        <template v-for="(module, moduleIndex) in page">
	    <!--echarts 图表-->
            <div v-if="isChartModule(module)" :key="moduleIndex + '-1'" :class="getModuleClasses(module)">
              <div class="chartBox">
                <dofar-charts
                  :chart-data="moduleChartMap[getChartModuleKey(module)][0]"
                  :width="moduleChartMap[getChartModuleKey(module)][1]"
                  :height="moduleChartMap[getChartModuleKey(module)][2]"
                />
              </div>
            </div>
            <!--普通 DOM 结构-->
            <div v-else :key="moduleIndex + '-2'" v-html="module.outerHTML"/>
        </template>
	</div>
</div>
```

上面的代码中用到了两个方法

 `isChartModule(module)` ：已知图表类型的模块都有 `echarts-module` 类，可以以此判断是否是图表类型

`getChartModuleKey(module)` ：明确了确实是图表类型的话，就可以从类名中取出 `moduleId`



#### 5 其他类型处理？

第四步单独处理了图表类型，由此可以推断，假如日后有其他既不是图表，也不是普通 `DOM` 结构类型的也需要单独处理，步骤与第四步类似。



#### 6. 修正 `selectedReportModuleIds`

最开始 `selectedReportModuleIds` 保存的是勾选的指标 `id` ，检查页面时发现假如只选中了三级指标，那么其所属的二级、一级指标不会显示，这显然不符合要求。

所以 `selectedReportModuleIds` 应该包含两部分：全选 + 半选中

```javascript
// 全选 moduleIds
this.checkedReportModuleIds = this.$refs.moduleTree?.getCheckedKeys?.() ?? []

// 半选 moduleIds
this.halfCheckedReportModuleIds = this.$refs.moduleTree?.getHalfCheckedKeys?.() ?? []

// 计算属性
selectedReportModuleIds() {
    return (this.checkedReportModuleIds.concat(this.halfCheckedReportModuleIds))
}
```



#### 7. 排版后的报告序号动态更新

已知需求要求每一层指标都需要有序号，类似 `1.`、 `1.1.`、 `1.1.1.`，且选择部分指标重新排版后序号需要动态更新。

已知 `selectedReportModuleIds`（全选 + 半选 `moduleIds`）和 `reportModules` （全部节点树），可以得到全选和半选包含的节点树 `selectedReportModules`。
```javascript
// 计算属性
selectedReportModules() {
  // 过滤出全选和半选包含的节点树
  const recurse = (module) => {
    const children = (module.children?.length > 0) ? (module.children.map(recurse).filter(Boolean)) : []

    const keepSelf = this.selectedReportModuleIds.includes(module.id)

    if (!keepSelf && children.length === 0) return null

    // 返回新节点（浅拷贝原始字段，children 为过滤后的 children）
    const newModule = {...module}
    if (children.length > 0) {
      newModule.children = children
    } else {
      // 如果没有子节点可删除 children 字段
      delete newModule.children
    }

    return newModule
  }
  
  const filteredModules = this.reportModules.map(recurse).filter(Boolean)

  
  // 过滤的节点树添加序号
  const addSeq = (modules, prefix = '') => {
    if (!Array.isArray(modules)) return

    modules.forEach((module, idx) => {
      let seq = ''
      if (module.id !== 'all') {
        seq = prefix ? `${prefix}${idx + 1}.` : `${idx + 1}.`
      }
      module.seq = seq

      if (Array.isArray(module.children) && module.children.length > 0) {
        // 递归处理子节点
        addSeq(module.children, seq)
      }
    })
  }
  
  addSeq(filteredModules, '')

  return filteredModules
}
```

拿到 `selectedReportModules` 后递归遍历便可以得到每个 `selectedModuleId` 对应的动态序号。



#### 8. 其他

1. 报告每页需要保持 `A4` 尺寸比例，且在不同屏幕宽度时需要自适应，定义如下三个基础变量

   ```javascript
   data() {
       return {
           pageWidthRem: '3.099rem',
         	pageVerticalPaddingRem: '0.4167rem',
         	a4Ratio: 297 / 210,  // A4 高:宽 比例（固定常量）
       }
   }
   ```

   `data` 中定义的变量可以在 `template` 、`script` 内使用，要想让其也可以在 `style` 内使用，需要将变量映射为 CSS 自定义属性

   ```javascript
   // 计算属性
   cssVars() {
       return {
           '--page-width-rem': this.pageWidthRem,
           '--a4-ratio': this.a4Ratio,
           '--page-vertical-padding-rem': this.pageVerticalPaddingRem
       }
   }
   ```

   将自定义属性绑定到根元素

   ```vue
   <template>
       <div :style="cssVars">
       	...
       </div>
   </template>
   ```

   `style` 内用类似 `var(--page-width-rem)` 的写法便可使用



2. 何时进行重新排版操作？

   - 第一次调用接口拿到分析报告内容后

   - 打开弹窗选择报告指标，点击确定后

   - 任何影响报告内容的异步操作后

     例如报告中有一块展示的思维导图形式的内容，之前的同事先使用 `jsMind` 初始化渲染，完毕后又将其转成了一张图片放入 `DOM`，此时就需要调用重新排版的方法了
