之前负责的网络中控平台里面有个功能：终端升级。顾名思义，就是在平台上传不同型号的固件包、软件包等，最终下发到教室更新。

有天测试同学联系到我，说上传包完成后报错了去看看。OK，`F12` 打开调试工具，就看到了一条 `error` 信息：

`getFileHash: NotReadableError`

`The requested file could not be read, typically due to permission problems that have occurred after a reference to a file was acquired.`

初看好像是没有权限访问，无法读取文件，但是细想一下，只有这个文件上传成功后获取文件 `md5` 值报错，其他文件都是成功状态，猜测问题应该出在这个文件上面。将现有拿到的信息去问 `ChatGPT`， `AI` 给出的常见原因有：

1. 用户权限或文件正在被占用
2. `File/Blob` 对象已经失效或不是合法的 `File`
3. 浏览器安全策略（比如在某些上下文或隐私模式下）
4. 读入方式或读取时机不当
5. 浏览器兼容或实现问题
6. 文件大小或资源受限

咦，文件大小也有影响？！好吧，当即去看看文件大小有 `3G` 多，再去看看 `getFileHash` 的实现

```vue
import md5 from 'js-md5'

<script>
    getFileHash(file) {
      return new Promise((resolve, reject) => {
        const fileReaderInstance = new FileReader()

        fileReaderInstance.onload = () => {
          const fileData = fileReaderInstance.result
          const fileHash = md5(fileData)
          resolve(fileHash)
        }

        fileReaderInstance.onerror = () => {
          reject(fileReaderInstance.error)
        }

        fileReaderInstance.readAsArrayBuffer(file)
      })
    }
</script>
```

猜测因为文件大小过大，所以一次性读取文件报错了。

幸好，`AI` 也给出了解决方法：

> 尝试读取较小文件验证；如果大型文件，考虑按分片读取（`file.slice`）并对每片计算哈希（例如采用 `spark-md5` 分片增量哈希），而不是一次性把整个文件读入内存



因此调整计算文件 `MD5` 值的方法：

```javascript
import SparkMD5 from 'spark-md5'

// 计算文件 MD5（分片增量方式，适合大文件）
export function getFileHashWithChunk(file, options = {}) {
  const {
    chunkSize = 4 * 1024 * 1024, // 每片 4MB
  } = options

  return new Promise((resolve, reject) => {
    const fileSize = file.size
    // 我们统一使用分片方案（对大文件必需，对小文件也更稳健）
    const chunks = Math.ceil(fileSize / chunkSize)
    const spark = new SparkMD5.ArrayBuffer()
    const fileReader = new FileReader()
    let currentChunk = 0

    // 单片读取
    function readChunk(startChunk) {
      const start = startChunk * chunkSize
      const end = Math.min(start + chunkSize, fileSize)
      const blobSlice = file.slice(start, end)

      fileReader.onload = (e) => {
        const arrayBuffer = e.target.result
        spark.append(arrayBuffer)
        currentChunk++
        
        if (currentChunk < chunks) {
          // 读取下一片
          readChunk(currentChunk)
        } else {
          // 全部读取完毕，返回 hash
          try {
            const hash = spark.end()
            resolve(hash)
          }
        }
      }

      fileReader.onerror = () => { ... }

      fileReader.readAsArrayBuffer(blobSlice)
    }

    // 启动读取
    try {
      if (fileSize === 0) {
        // 空文件的 md5
        resolve(spark.end())
      } else {
        readChunk(0)
      }
    }
  })
}
```

启动运行发现报错消失了，完美解决！



既然对于大文件（项目里预设 `100M`）有分片哈希，那么猜测应该也有分片上传
